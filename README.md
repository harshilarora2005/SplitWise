# Splitwise Clone

A full-stack expense splitting application built with **React + Vite** (frontend) and **Spring Boot** (backend), using **PostgreSQL** as the database.

Split expenses with friends, track group spending, and automatically calculate the minimum transactions needed to settle all debts.

---

## Features

- **Authentication** вЂ” JWT-based register/login
- **Groups** вЂ” Create groups, add members, leave or delete groups
- **Expenses** вЂ” Add expenses with equal or custom splits, categorize them, assign who paid
- **Settlements** вЂ” Auto-calculated minimum transaction plan to settle all debts (debt simplification algorithm)
- **Dashboard** вЂ” Overview of total spend, your share, category breakdowns, and recent expenses

## Prerequisites

- **Java 21**
- **Maven 3.9+** (or use the included `./mvnw` wrapper)
- **Node.js 20+** and **npm**
- **PostgreSQL 14+**

---

## Database Setup

Create the database before starting the backend:

```sql
CREATE DATABASE settleup;
```

The schema is auto-generated by Hibernate (`spring.jpa.hibernate.ddl-auto=update`), so no migrations are needed.

---

## Backend Setup

### 1. Configure `application.properties`

Edit `split/src/main/resources/application.properties`:

```properties
spring.application.name=split

# PostgreSQL connection
spring.datasource.url=jdbc:postgresql://localhost:5432/settleup
spring.datasource.username=YOUR_POSTGRES_USERNAME
spring.datasource.password=YOUR_POSTGRES_PASSWORD

# JPA / Hibernate
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect
spring.jpa.properties.hibernate.format_sql=true

# Error messages
server.error.include-message=always

# JWT
jwt.secret=mySuperSecretJwtKeyForSplitwiseProject12345
jwt.expiration=86400000
```

> **Note:** `jwt.secret` must be at least 32 characters. `jwt.expiration` is in milliseconds (86400000 = 24 hours). In production, store secrets via environment variables, not in the properties file.

### 2. Run the backend

```bash
cd split

# Using the Maven wrapper (recommended)
./mvnw spring-boot:run

# Or with system Maven
mvn spring-boot:run
```

The backend starts on **http://localhost:8080**.

## Frontend Setup

### 1. Install dependencies

```bash
cd frontend
npm install
```

### 2. Run the dev server

```bash
npm start
```

The frontend starts on **http://localhost:5173** and proxies all `/api` requests to `http://localhost:8080` (configured in `vite.config.js`).

---

## Running the Full Stack

1. Start PostgreSQL
2. Start the Spring Boot backend (`./mvnw spring-boot:run` inside `split/`)
3. Start the Vite dev server (`npm start` inside `frontend/`)
4. Open **http://localhost:5173** in your browser

## API Overview

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/api/auth/register` | Register a new user |
| `POST` | `/api/auth/login` | Login and get JWT |
| `GET` | `/api/users` | List all users |
| `GET` | `/api/users/me` | Get current user |
| `POST` | `/api/groups/create` | Create a group |
| `GET` | `/api/groups` | Get your groups |
| `GET` | `/api/groups/{id}` | Get group details |
| `DELETE` | `/api/groups/{id}/leave` | Leave a group |
| `DELETE` | `/api/groups/{id}` | Delete a group (creator only) |
| `POST` | `/api/expenses/add` | Add an expense |
| `GET` | `/api/expenses/group/{groupId}` | Get expenses for a group |
| `GET` | `/api/expenses/all` | Get all your expenses |
| `DELETE` | `/api/expenses/{id}` | Delete an expense |
| `GET` | `/api/settlements/group/{groupId}` | Get settlement plan |
| `POST` | `/api/settlements/record` | Record a settlement |

All endpoints except `/api/auth/**` require a `Bearer <token>` header.

## Environment Variables (Production)

Instead of hardcoding credentials in `application.properties`, use environment variables:

```properties
spring.datasource.username=${DB_USERNAME}
spring.datasource.password=${DB_PASSWORD}
jwt.secret=${JWT_SECRET}
```

Then set them when running the jar:

```bash
DB_USERNAME=postgres DB_PASSWORD=secret JWT_SECRET=your-secret ./split.jar
```
